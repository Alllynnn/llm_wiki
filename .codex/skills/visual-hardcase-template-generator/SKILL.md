# Visual Hardcase Template Generator

## Core Rule

Generate templates from retrieved LLM Wiki rules, not from generic visual-question patterns. Preserve source constraints and make the generated template reproducible.

This retrieval rule must travel with the distributed skill. Do not generate task-type-specific templates from memory or stale chat context; retrieve LLM Wiki evidence first and mark unavailable evidence as `待知识库确认`.

## Workflow

1. Search LLM Wiki for the requested task type and construction rules:
   `python scripts/wiki_search.py "<题型> 构造 规则 模板" --top-k 10 --include-content`
2. Search bad-case records for the same task type:
   `python scripts/wiki_search.py "<题型> bad case 复盘" --top-k 8 --include-content`
3. Read:
   - `references/hermes-template-principles.md`
   - `references/template-contract.md`
4. Generate a fielded template with source basis, prompt pattern, answer contract, validation checklist, and common bad-case traps.
5. Include citations to wiki titles and paths used to shape the template.

If the API is unavailable, produce only a skeleton marked `待知识库确认`, not a final task-type-specific template.

## Newcomer Training Template

When the user asks for newcomer onboarding or a local training table, hand off to `visual-hardcase-training-review` generate mode. This skill supplies the instructional content: project workflow, task-type quick reference, prompt/answer template fields, and self-check list. The workbook itself is generated by `scripts/generate_training_workbook.py` in the training-review skill.
