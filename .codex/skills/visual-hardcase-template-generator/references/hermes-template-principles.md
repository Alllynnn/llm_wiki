# Hermes-Style Template Principles

The Hermes reference pattern is structural, not visual styling:

- Use frontmatter or fixed metadata fields before the body.
- Separate source evidence, generated content, and checking instructions.
- Preserve source constraints exactly; do not add unsupported examples.
- Make prompts reproducible by writing the full prompt pattern and required variables.
- End with a quality checklist that can be executed by another reviewer.

Apply these principles to hardcase construction templates:

1. Start with task identity and source basis.
2. Define required input fields.
3. Give a prompt skeleton with placeholders.
4. Define answer format and validation method.
5. List task-specific bad-case traps from LLM Wiki evidence.
6. Add a self-check list suitable for pre-QC.
